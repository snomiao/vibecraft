# Glossary

- Game: The app/product, used conversationally even though it is not strictly a game.
- World (Workspace): The in-app world tied to a root directory; "world" and "workspace" are used interchangeably.
- Map (Game Area, Canvas): The main world surface where entities live and interact.
- HUD: Overlay UI; includes a Top HUD (header) and Bottom HUD (details/actions).
- Top HUD: Upper overlay with global controls.
- Bottom HUD: Lower overlay with entity details and actions.
- Unit: Controllable agent entity.
- Hero: Special unit with privileges; cannot attach to buildings.
- Building: World-fixed interactable (e.g., Claude Code, folder).
- Folder: A building tied to a folder on your computer.
- Item: Inventory UI entry (placeholder).
- Attach: Order for units to orbit a building after reaching its hitbox. Heroes cannot attach. Provider buildings (Claude Code, ChatGPT Codex) do not accept attachments; units will move to the building and stop instead.
- Orbit: Visual movement of attached units around a building.
- Saved layout data: Per‑workspace positions, names, and links for your buildings.
- Subscription: Paid access record tied to a device; active or trialing subscriptions unlock the app.
- Trial: Seven-day access window that begins on the first successful license registration.
- Pairing code: Short-lived code used to link an additional device to an existing subscription.
